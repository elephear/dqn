## src目录代码实现分析

基于对 `src/`目录下源代码的详细审查，以下说明代码如何实现 `full_updated.md`中第4章“基于GCN-DQN的算力感知调度算法”的各个模块。

### 1. 算法总体框架（对应4.1节）

* **核心类** ：`GCN_DQN_Scheduler`（位于 `src/models/scheduler.py`）整合了五个模块：
* 状态感知模块：通过 `network_state`字典实时收集网络拓扑、链路状态和算力资源信息。
* 特征提取模块：`GCNFeatureExtractor`（`src/models/gcn.py`）从联合状态张量中提取拓扑特征。
* 决策优化模块：`DQNAgent`（`src/models/dqn.py`）基于特征进行路径-节点决策。
* 动态调整模块：`DynamicWeightAdjuster`（`src/algorithms/weight_adjuster.py`）根据业务类型和网络状态调整多目标权重。
* 缓存加速模块：`FastPathCache`（`src/algorithms/path_cache.py`）通过LRU缓存提升决策效率。

### 2. 图卷积网络特征提取（对应4.2节）

* **实现文件** ：`src/models/gcn.py`
* **关键组件** ：
* `GCNLayer`：实现单层图卷积运算 H=σ(AXW)**H**=**σ**(**A**X**W**)，支持添加自连接和归一化邻接矩阵。
* `GCNFeatureExtractor`：两层GCN架构，第一层将节点特征（计算能力、负载、能耗系数等）映射到隐藏维度，第二层输出节点级特征。
* 特征聚合：采用均值池化（`torch.mean`）将节点级特征聚合为图级特征向量 hG**h**G。
* **与论文一致性** ：完全遵循论文4.2节的数学公式，包括归一化邻接矩阵 D−1/2AD−1/2**D**−**1/2**A**D**−**1/2** 和ReLU激活函数。

### 3. 深度Q网络决策优化（对应4.3节）

* **实现文件** ：`src/models/dqn.py`
* **关键组件** ：
* `QNetwork`：双流架构，状态流处理图级特征，动作流处理动作嵌入（目标节点和下一跳的one-hot编码），融合后输出Q值。
* `DQNAgent`：实现经验回放、目标网络、ε-greedy探索策略和Double DQN训练。
* 奖励函数：在 `scheduler.py`中根据业务类型（边缘AI推理或算力跨域调度）设计差异化奖励。
* **与论文一致性** ：
* 采用Huber损失函数，目标网络定期更新。
* 动作空间包含两个维度：算力节点选择和下一跳选择。
* ε-greedy策略的ε随时间衰减。

### 4. 动态权重调整机制（对应4.4节）

* **实现文件** ：`src/algorithms/weight_adjuster.py`
* **关键组件** ：
* `DynamicWeightAdjuster`：维护两类业务的标准权重（边缘AI：[1.0,0,0]；算力跨域调度：[0.4,0.3,0.3]）。
* 实时调整：根据网络状态计算紧迫性因子 ϕk(st)**ϕ**k(**s**t)，动态调整权重 wk′**w**k**′**。
* 平滑过渡：当业务类型切换时，权重以过渡速率 β=0.1**β**=**0.1** 平滑过渡。
* **与论文一致性** ：完全实现论文4.4节的权重计算公式，包括紧迫性因子和归一化处理。

### 5. 快速路径缓存技术（对应4.5节）

* **实现文件** ：`src/algorithms/path_cache.py`
* **关键组件** ：
* `FastPathCache`：基于LRU的缓存，键为（源节点、目的节点、网络状态哈希），值为（目标节点、路径、Q值）。
* 缓存验证：每次命中时验证缓存决策的可行性（带宽、时延约束）。
* 失效机制：当网络状态变化超过阈值时，相关缓存条目失效。
* **与论文一致性** ：实现两级缓存（节点对缓存和状态缓存），支持缓存预热和LRU替换策略。

### 6. 算法训练与部署流程（对应4.6节）

* **实现文件** ：`src/models/scheduler.py`、`src/experiments/experiment_runner.py`
* **训练流程** ：

1. 离线训练：在仿真环境（`src/environment/simulation_env.py`）中收集经验数据。
2. 交替训练：GCN和DQN联合训练，每轮迭代包含前向传播、经验回放和参数更新。
3. 超参数配置：学习率0.001、折扣因子0.99、回放缓冲区大小10000等，与论文4.6.3节一致。

* **在线部署** ：
* 状态感知 → 特征提取 → DQN决策 → 缓存查询 → 执行调度 → 反馈循环。

#### 11.4 训练GCN-DQN调度器

**单次训练运行**：
```python
from src.models.scheduler import GCN_DQN_Scheduler
from src.environment.simulation_env import SimulationEnvironment
from src.utils.config_loader import load_config

# 加载配置
config = load_config("config/config.yaml")

# 创建调度器和环境
scheduler = GCN_DQN_Scheduler(config)
env = SimulationEnvironment(config)

# 设置环境（30节点网络）
env.setup(network_size=30, initial_requests=200)

# 训练模式
scheduler.train()

# 运行训练循环
for episode in range(1000):
    state = env.reset()
    total_reward = 0
    
    for step in range(100):
        # 获取业务请求
        request = env.generate_request()
        
        # 调度决策
        decision = scheduler.schedule(request, env.get_network_state())
        
        # 执行决策并获取奖励
        reward, done = env.step(decision)
        total_reward += reward
        
        # 收集经验
        scheduler._collect_experience(request, decision, reward, done)
        
        if done:
            break
    
    # 每100轮进行一次训练
    if episode % 100 == 0:
        scheduler.train_step(batch_size=32)
    
    print(f"Episode {episode}: Total Reward = {total_reward:.2f}")
```

##### 11.8.2 配置说明
配置文件 `config/config.yaml` 已添加设备配置节：
```yaml
# 设备配置
device:
  type: "auto"  # auto, cuda, cpu
  device_id: 0  # GPU设备ID
```

**配置选项**：
- `type: "auto"`：自动检测并使用GPU（如果可用），否则使用CPU
- `type: "cuda"`：强制使用GPU（如果不可用会报错）
- `type: "cpu"`：强制使用CPU
- `device_id: 0`：指定使用的GPU设备ID（多GPU系统）

#### 11.9 注意事项

1. **首次运行**：首次运行时会生成网络拓扑和业务数据，可能需要较长时间。
2. **GPU加速**：如果系统有GPU，PyTorch会自动使用GPU加速训练。
3. **内存管理**：大规模网络（100+节点）可能需要更多内存，建议调整 `config.yaml` 中的参数。
4. **结果复现**：设置随机种子以确保结果可复现：
   ```python
   import torch
   import numpy as np
   import random
   
   torch.manual_seed(42)
   np.random.seed(42)
   random.seed(42)
   ```

#### 11.9 故障排除

1. **导入错误**：确保项目根目录在Python路径中：
   ```python
   import sys
   sys.path.append(".")
   ```

2. **配置错误**：检查 `config/config.yaml` 文件格式是否正确。

3. **内存不足**：减少 `batch_size` 或 `replay_buffer_size`。

4. **训练不稳定**：调整学习率或增加 `target_update_freq`。
